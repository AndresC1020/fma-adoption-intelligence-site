Las tres vistas del tablero

¿Qué responde cada vista?

Cada vista aborda un nivel distinto de análisis. Las tres son complementarias: la descriptiva explica el pasado, la predictiva anticipa el futuro y la prescriptiva orienta la acción.

V1
Factores de adopción
Análisis descriptivo

Responde: ¿qué comportamientos en los primeros 14 días distinguen a un técnico que retendrá FMA de uno que la abandonará? Los 10 factores estadísticamente significativos (p < 0.05) se presentan ordenados por magnitud de efecto con la comparación directa entre grupo adoptante y grupo abandono.

  • Filtra por industria, tamaño de cliente o plataforma de dispositivo para analizar segmentos específicos.
  • La tabla comparativa muestra promedios y p-valor de cada variable.
  • El gráfico de funcionalidades identifica qué herramientas (Job Close Out, Time, Payment) marcan la diferencia en la primera semana.
  • El ranking SHAP global ordena las 10 variables con mayor impacto en el modelo predictivo.
V2
Puntuación de riesgo
Análisis predictivo

Responde: ¿qué clientes tienen mayor riesgo de que sus técnicos no retengan FMA? La tabla principal ordena clientes de mayor a menor riesgo. El score de cada cliente es el promedio ponderado de los scores individuales de sus técnicos.

  • Score por cliente en escala 0-100%: valores bajos indican mayor riesgo de abandono.
  • Filtra por nivel de riesgo (Crítico, Alto, Medio, Bajo) para concentrar la vista en los casos más urgentes.
  • Selecciona un cliente para ver el detalle individual de cada técnico con su score y sus 3 factores SHAP determinantes.
  • La columna de acción sugerida resume la recomendación operativa por cliente.
V3
Guía de migración
Análisis prescriptivo

Responde: ¿qué acción concreta debe tomar el PM con cada cliente? Los clientes de la muestra se distribuyen en cuatro segmentos según el porcentaje de sus técnicos activos en FMA. Cada segmento tiene acciones diferenciadas respaldadas por el modelo. El conteo exacto por segmento se calcula dinámicamente en el tablero a partir del archivo tenant_scores.parquet y se actualiza con los filtros aplicados.

  • Crítico (<20% en FMA): intervención inmediata, acompañamiento activo primeros 30 días.
  • A activar (20-39%): activar funcionalidades clave según SHAP dominante de cada técnico; objetivo: superar el umbral del 40% en 4 semanas.
  • A monitorear (40-69%): mantener ritmo, recuperar técnicos rezagados con score 30-50%.
  • En buen camino (≥70%): seguimiento estándar, sin tarjeta de intervención.
Código fuente

Repositorio GitHub

Código completo del pipeline de datos, modelo predictivo y tablero Streamlit.

Ver repositorio en GitHub ↗

app.py

Aplicación principal de Streamlit. Implementa las tres vistas analíticas (Factores, Riesgo, Guía) y la lógica de navegación entre ellas. Lee los archivos Parquet generados por el pipeline y expone los controles de filtrado interactivo. Desplegado en Streamlit Community Cloud, sin instalación local para el usuario final.

pipeline.py

Núcleo del sistema. Ejecuta cinco fases secuenciales: ETL (limpieza y unificación de fuentes), ingeniería de características (57 variables por técnico en ventana de 14 días), scoring con Random Forest, cálculo de SHAP values y agregación por cliente. Tiempo total: ~21 seg sobre la muestra académica y 8-14 min en producción con ~5K técnicos.

src/

Módulos de soporte encapsulados para mantener el código organizado y reutilizable. etl.py contiene todas las transformaciones de datos y la construcción del feature store; config.py centraliza las rutas de archivos y los parámetros del sistema. Esta separación permite cambiar fuentes de datos o rutas sin modificar el pipeline principal.

Ejecución local

Cómo correr el proyecto

Requiere Python 3.9+. Los datos de muestra están incluidos en el repositorio (n=140 técnicos, 54 clientes).

# 1. Clonar el repositorio
git clone https://github.com/mbayonal/fma-adoption-intelligence.git

# 2. Crear y activar entorno virtual
python -m venv .venv
source .venv/bin/activate # macOS/Linux
.venv\Scripts\Activate.ps1 # Windows PowerShell

# 3. Instalar dependencias
pip install -r requirements.txt

# 4. Generar datos analíticos (~21 seg con datos de muestra)
python pipeline.py

# 5. Lanzar el tablero
streamlit run app.py
# Abre en http://localhost:8501
Nota: El tiempo del pipeline en producción (dataset completo ~5K técnicos) es de 8-14 minutos según el hardware disponible. El manual de usuario detalla los requisitos del sistema y el proceso completo. Ver documentación →
