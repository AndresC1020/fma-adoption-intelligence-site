Arquitectura del pipeline

Flujo completo desde los datos crudos hasta el tablero interactivo. Cada componente tiene una responsabilidad única y bien definida.

1. Fuente de datos
Datos de origen
  • Snowflake (producción) / CSV muestra local
  • FullStory: 700K+ eventos de comportamiento en campo
  • 140 técnicos, 54 clientes (muestra de validación académica)
2. Procesamiento central
pipeline.py
  • ETL: limpieza y unificación de tenants, técnicos y eventos
  • Features: 57 variables por técnico (ventana de 14 días)
  • Scoring: Random Forest predice P(retención) [0,1] por técnico
  • SHAP: top-3 factores explicativos individuales por técnico
  • Agregación: score medio y segmento por cliente
~21 seg (muestra n=140)  ·  8-14 min (producción ~5K técnicos)
3. Almacenamiento intermedio
/data/*.parquet
  • analytical_dataset — features + score + nivel de riesgo por técnico
  • shap_values — top-3 factores explicativos por técnico
  • feature_importance — ranking global de variables del modelo
  • tenant_scores — score agregado y segmento por cliente
4. Aplicación interactiva
app.py, Streamlit
  • V1 Factores: análisis descriptivo del comportamiento en primeros 14 días
  • V2 Riesgo: score predictivo por cliente y desglose SHAP por técnico
  • V3 Guía: segmentación prescriptiva con acciones diferenciadas por segmento
5. Usuario final
Tablero en el navegador
  • Streamlit Community Cloud — acceso sin instalación local
  • Product Manager y stakeholders de ServiceTitan
  • Ruta a producción: migración a Tableau + Snowflake
Flujo de datos: Los datos crudos (CSV o Snowflake) son procesados por pipeline.py, que genera cuatro archivos Parquet con scores y factores SHAP. La aplicación app.py lee esos archivos y los presenta en el tablero interactivo. Ruta a producción: Migración a Tableau + Snowflake. Extracción bisemanal (~5.000 técnicos activos). Re-entrenamiento estimado: 8-14 min sobre dataset completo.
Tecnologías y dependencias

Stack técnico

El sistema usa exclusivamente librerías de código abierto bajo licencias permisivas (MIT, BSD). No requiere licencias de software comercial ni infraestructura adicional más allá de Python 3.9+ y un navegador web.

Procesamiento de datos

pandas maneja la carga, limpieza y transformación de los datasets (tenants, técnicos, eventos). numpy se usa en los cálculos numéricos vectorizados del feature engineering. Juntas, estas dos librerías procesan los 700K+ registros de eventos FullStory y construyen las 57 variables por técnico de la ventana de observación de 14 días.

Modelo y explicabilidad

scikit-learn provee el Pipeline con preprocesamiento (ColumnTransformer + OneHotEncoder) y el RandomForestClassifier con class_weight="balanced" para compensar el desbalance de clases. shap (TreeExplainer) calcula los valores SHAP para cada técnico, generando explicaciones individualizadas del score. joblib serializa y deserializa el modelo entrenado para no re-entrenar en cada ejecución.

Tablero e infraestructura

Streamlit construye la interfaz interactiva de las tres vistas analíticas en Python puro, sin necesidad de HTML o JavaScript. Streamlit Community Cloud aloja el prototipo de forma gratuita con acceso público desde cualquier navegador. plotly genera las visualizaciones interactivas (gráficos de barras, scatter, series de tiempo) embebidas en las vistas.
